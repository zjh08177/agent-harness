# Key Outputs Writing Guide

## Core Principle

Key Outputs summarize **"what you did"** during the review cycle. Focus on:
- What work did you do?
- What role did you assume?
- What output did you achieve?
- What are the effects/impact?

## Quality Criteria

### Focused
Write about the most important outputs and contributions (usually no more than 5 items). Don't list every minor detail.

### Effective
Review and summarize with moderate granularity—not too general or too precise.

### Complete
- Explain results and impacts using **specific business data and progress**
- Explain your actual contribution based on the **role you assumed**
- This helps 360° reviewers fully understand your work

### Confidential
Since Key Outputs are visible to all 360° reviewers, desensitize confidential information using codewords or document access control.

## Template Structure

```
[Key Output N]

In [project/product/initiative], as [role/responsibility], [action taken], achieving [specific outcome].

Key metrics/results:
- [Metric 1]: [before] → [after] (context/attribution)
- [Metric 2]: [quantified result]
- [Impact statement]
```

## Good Examples

### Example 1: Algorithm/Technical Work
```
As the person in charge of the recommendation algorithm strategy of product AAA, made improvements in **model, **algorithm, etc. Key indicators:
- DAU **=> **
- Penetration rate *% => *%, 1-day retention *% => *% (effects brought by **algorithm improvement)
- User satisfaction slightly increased *% => *%
```

### Example 2: System/Platform Work
```
Based on ** requirements, as [job responsibility], proposed BBB system planning for the first time, promoted to build and launch the product from 0 to 1, systematized the online unified management of **.
① Scope: [coverage], improving [efficiency metric] compared to previous methods
② Data: [data flow/visibility improvements]
③ Business value: [commercial/strategic impact]
```

### Example 3: Team/Process Work
```
Team management
- Integrated team ** and ** for more reasonable division of labor
- Solved [specific problem] in [area]
```

## Common Mistakes to Avoid

| Wrong | Right |
|-------|-------|
| List outputs of projects you barely participated in | Describe the role you assumed to demonstrate actual contribution |
| List items without any output | Include specific data, results, and impact |
| Write vague/generic descriptions | Include concrete metrics, timelines, and outcomes |
| Include every small task | Focus on 3-5 highest-impact items |
