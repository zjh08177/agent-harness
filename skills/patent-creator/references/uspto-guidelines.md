# USPTO Guidelines for Software & AIGC Patents

## Core Patentability Requirements

### Three Fundamental Criteria
1. **Novelty**: Invention must be genuinely new
2. **Non-obviousness**: Not a logical extension of existing methods to one skilled in the art
3. **Utility**: Must have practical, functional application

## Software Patent Eligibility (Post-Alice)

### Avoiding "Abstract Idea" Rejections

Software patents must transcend abstract ideas. To establish eligibility:

1. **Demonstrate Technical Effect**: Show tangible improvement in technology
   - Improved computing speeds
   - Reduced computing resources required
   - Unconventional way of performing computing tasks

2. **Anchor in Specific Technology**: Tie your architecture to specific technological applications

3. **Include Hardware Implementation**: Specific hardware implementations help establish eligibility

### What Gets Rejected
- Claims directed only at high-level training and use of trained models
- Abstract concepts without inventive implementation
- Software that fails to provide technological improvement

## AI/AIGC Patent Specific Guidance (USPTO 2024-2025)

### Inventorship Rules
- **AI cannot be an inventor** - only humans can be listed as inventors
- Using AI tools during development does NOT forfeit human inventorship
- AI is treated like any other tool (laboratory equipment, software, databases)
- Applications listing AI as inventor will be rejected under 35 USC 101 and 115

### AI Patent Eligibility Tips
- Explicit explanation of technological improvement doesn't need to be in specification
- Improvement must be apparent to one skilled in the art from the description
- Focus on specific implementations, not just high-level AI concepts

## Required Documentation Checklist

### Mandatory Documents

| Document | Description |
|----------|-------------|
| **Transmittal Form** | Form PTO/AIA/15 identifying application contents |
| **Specification** | Detailed written description enabling implementation |
| **Claims** | At least one claim defining scope of protection |
| **Abstract** | ~150 words summarizing the invention |
| **Drawings** | Required if necessary to understand the invention |
| **Oath/Declaration** | Each inventor signs confirming original inventorship |
| **Filing Fee** | Varies by entity size |

### Recommended for Software Architecture Patents

**Minimum 5 patent drawings:**
1. System architecture/network diagram
2. Main method flowchart
3. Additional method variation flowcharts
4. Block diagram of hardware elements
5. Computer system/server diagram

## Claims Drafting Best Practices

### 5-Step Process
1. Understand innovation as problem-solution approach
2. Prepare flowcharts covering all process aspects
3. Prepare system architecture (block diagrams) with all hardware elements
4. Ensure sync between flowcharts and block diagrams
5. Draft both system/apparatus AND method claims

### Avoiding Common Issues

**Divided Infringement**:
- Draft claims assigning all steps to a SINGLE entity
- Avoid splitting actions between different parties

**Future-Proofing**:
- Consider potential future technology developments
- Draft claims broad enough to encompass future iterations

### Claim Types to Include
1. **Independent claims**: Broad coverage of main invention
2. **Dependent claims**: Specific implementations and variations
3. **Method claims**: Process/steps
4. **System/apparatus claims**: Components and structure

## Software Architecture Documentation

### What to Include in Specification

1. **Technical Architecture**
   - Client-server model
   - Data flow
   - Components and their interactions

2. **Functionality Description**
   - From user's perspective
   - From machine's perspective

3. **Best Mode**: Your preferred implementation approach

### Required Information for Software Patents
1. General description of computer architecture
2. Single flowchart showing overall software operation
3. Series of flowcharts for innovative features implementation

## Timeline and Process

- **Processing time**: Typically 2-3 years for software patents
- **USPTO Streamlined Pilot**: For applications with 1 independent claim, 10 total claims max
- **Filing format**: DOCX recommended (avoids $400 surcharge)

## Common Rejection Reasons

1. **Abstract Idea (35 USC 101)**: Claims directed to abstract concept without inventive concept
2. **Lack of Technical Contribution**: Software fails to provide technological improvement
3. **Insufficient Description (35 USC 112)**: Not enough detail for skilled person to implement
4. **Prior Art (35 USC 102/103)**: Invention anticipated or obvious over existing art
